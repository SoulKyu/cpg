          1. **Recommended for ad-hoc bootstrap:** the `policy.cilium.io/proxy-visibility` annotation on the workload (deprecated upstream but still widely supported as of Cilium 1.19).           2. **Recommended for permanent enforcement:** ship a starter L7 CNP (snippet below) that triggers Envoy proxy injection without enforcing rules. The very act of having an L7 rule on a workload makes Cilium proxy that workload's traffic.
          3. **Cluster-wide:** `enable-l7-proxy: true` in the cilium-config ConfigMap (default true on most installs; required for any of the above to work). cpg's pre-flight check catches this.

       d. **Starter L7-visibility CNP snippet** (copy-pasteable, valid Cilium YAML, fill-in comments use `<workload-name>` style placeholders annotated with `# replace`):
          ```yaml

          kind: CiliumNetworkPolicy
          metadata:
            name: cpg-l7-visibility-bootstrap
            namespace: production    # replace with your namespace
          spec:
            endpointSelector:
              matchLabels:
                app.kubernetes.io/name: my-app    # replace with your workload's label
            egress:
              # match-all HTTP rule triggers Envoy without enforcing a path/method.
              - toEndpoints:
                  - {}
                toPorts:
                  - ports:
                      - {port: "80", protocol: TCP}
                    rules:
                      http:
                        - {}
              # match-all DNS rule triggers DNS proxy visibility for kube-dns.
              - toEndpoints:
                  - matchLabels:
                      k8s-app: kube-dns
                      io.kubernetes.pod.namespace: kube-system
                toPorts:
                  - ports:
                      - {port: "53", protocol: UDP}
                      - {port: "53", protocol: TCP}
                    rules:
                      dns:
                        - matchPattern: "*"
          ```
