  <name>Task 1: Policy builder with test fixtures and TDD</name>
  <files>pkg/policy/testdata/ingress_flow.go, pkg/policy/builder.go, pkg/policy/builder_test.go</files>
  <behavior>
    - BuildPolicy(namespace, workload, []flow.Flow) -> *ciliumv2.CiliumNetworkPolicy
    - INGRESS dropped flow: endpointSelector = destination labels, IngressRule with FromEndpoints = source labels, ToPorts = destination port/protocol
    - EGRESS dropped flow: endpointSelector = source labels, EgressRule with ToEndpoints = destination labels, ToPorts = destination port/protocol
    - Mixed flows (both ingress and egress for same workload): policy has both Ingress and Egress rules
    - TypeMeta: APIVersion "cilium.io/v2", Kind "CiliumNetworkPolicy"
    - ObjectMeta: Name "cpg-{workload}", Namespace from arg, Labels {"app.kubernetes.io/managed-by": "cpg"}
    - Port extraction: flow.L4.GetTcp().DestinationPort -> Port as string, Protocol api.ProtoTCP
    - Port extraction: flow.L4.GetUdp().DestinationPort -> Port as string, Protocol api.ProtoUDP
    - Nil L4: flow is skipped (no rule generated, no panic)
    - Multiple flows with same peer but different ports: single rule with multiple ToPorts entries
    - Multiple flows with different peers: separate rules per peer
    - YAML serialization produces valid CiliumNetworkPolicy (roundtrip test)
  </behavior>
  <action>

   - `EgressUDPFlow(srcLabels, dstLabels []string, srcNs string, dstPort uint32)` -> egress UDP flow
   - `NilL4Flow()` -> flow with nil L4 (edge case)
   - Each helper sets `TrafficDirection`, `Source/Destination.Labels`, `Source/Destination.Namespace`, and `L4`

2. Write tests FIRST in `pkg/policy/builder_test.go` using testify/assert. Cover every behavior listed above:

   - TestBuildPolicy_MixedDirections: both ingress and egress rules in one policy
   - TestBuildPolicy_TypeMeta: correct APIVersion and Kind
   - TestBuildPolicy_ObjectMeta: correct name, namespace, managed-by label
   - TestBuildPolicy_NilL4: flow skipped, no panic, empty rules
   - TestBuildPolicy_SamePeerDifferentPorts: single rule with multiple ports
   - TestBuildPolicy_DifferentPeers: separate rules per peer
   - TestBuildPolicy_YAMLRoundtrip: marshal to YAML and verify structure
   Run tests (expect RED).

3. Implement `pkg/policy/builder.go`:
   - `type PolicyEvent struct { Namespace, Workload string; Policy *ciliumv2.CiliumNetworkPolicy }`
   - `func BuildPolicy(namespace, workload string, flows []*flow.Flow) *ciliumv2.CiliumNetworkPolicy`
   - Separate flows by TrafficDirection (INGRESS vs EGRESS)
   - For INGRESS: group by source labels -> build IngressRule per unique source
   - For EGRESS: group by destination labels -> build EgressRule per unique destination
   - Use `labels.BuildEndpointSelector()` for endpointSelector, `labels.BuildPeerSelector()` for peers

   - Port extraction: nil-check L4, then GetTcp()/GetUdp(), convert DestinationPort with `strconv.FormatUint`
   - Use `api.IngressCommonRule{FromEndpoints}` and `api.ToPorts` for ingress rules
   - Use `api.EgressCommonRule{ToEndpoints}` and `api.ToPorts` for egress rules
   - Use `Spec` (singular *api.Rule), NOT `Specs`
   Run tests (expect GREEN).
