<objective>
Add `--ignore-protocol` flag (repeatable, comma-separated, case-insensitive) on `cpg generate` and `cpg replay`. Drop matching flows EARLY in the aggregator — before bucketing, before BuildPolicy, before evidence — so excluded protocols produce no CNP rule, no evidence, and contribute to a transparent per-protocol counter logged in the session summary.

Purpose: Operators want to exclude noisy L4 protocols (typically ICMPv4/ICMPv6 denials from health probes) from generated policies without grepping output afterwards.

Output: Working flag end-to-end (live + replay), with TDD tests for filtering, flag parsing, validation, and an e2e replay assertion.
</objective>
