# Milestone v1.1 — Offline Replay & Policy Analysis

**Status:** ✅ Shipped
**Date:** 2026-04-24
**Phases:** 4 → 6
**Plans:** 3 total (1 + 1 + 1)

## Delivered

Offline iteration workflow for cpg: users can now replay previously captured Hubble jsonpb dumps through the same pipeline as live streaming, preview generation changes via `--dry-run` with a unified YAML diff, and inspect which flows produced each generated rule via `cpg explain`. Per-rule evidence is persisted to `$XDG_CACHE_HOME/cpg/evidence` and merges across sessions with FIFO caps.

## Key Accomplishments

1. **`cpg replay <file>`** — jsonpb line-delimited ingestion through `FlowSource` interface, transparent gzip and stdin (`-`), malformed/non-DROPPED counters surfaced in session summary (Phase 4).
2. **Per-rule attribution end-to-end** — `BuildPolicy` returns `[]RuleAttribution` with deterministic `RuleKey` encoding; threaded through `PolicyEvent` and aggregator (Phase 4).
3. **Evidence persistence** — JSON schema v1, XDG-aware paths hashed by output dir, atomic writes, reader with schema-version check, FIFO merge for samples/sessions (Phase 4).
4. **`--dry-run` mode** — policy and evidence writers become no-ops; unified YAML diff via `pkg/diff.UnifiedYAML` (ANSI on TTY); `--no-diff` suppresses diff only (Phase 5).
5. **Pipeline fan-out** — single `policies` channel tees into `policyCh` + `evidenceCh`; `evidenceWriter` goroutine independent of `policyWriter`; session session upsert in evidence merge (Phase 5).
6. **`cpg explain <target>`** — NAMESPACE/WORKLOAD or YAML path input; filters (`--ingress`/`--egress`, `--port`, `--peer KEY=VAL`, `--peer-cidr`, `--since`); text/JSON/YAML renderers; actionable error naming missing evidence path (Phase 6).
7. **Shared flags + README refresh** — `commonflags.go` factored; README quick-start, offline replay, dry-run, explain sections (Phase 6).

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| `FlowSource` interface promoted to `pkg/flowsource` | Decouples replay (file) from live (gRPC); testable without Hubble | ✓ Good |
| Evidence under `$XDG_CACHE_HOME/cpg/evidence`, keyed by hash of output dir | Multiple projects/output dirs don't collide; not committed; XDG-compliant | ✓ Good |
| Schema version pinned at 1 | Reader rejects unknown versions — safe forward-compat | ✓ Good |
| FIFO caps on samples/sessions | Bounded disk use; `--evidence-samples` / `--evidence-sessions` tunable | ✓ Good |
| Merge semantics mirror policy YAML (preserve rules not re-emitted) | Consistent mental model with on-disk policies | ✓ Good |
| Channel fan-out (tee) over shared handle | Policy writer and evidence writer independent; neither blocks the other | ✓ Good |
| `--dry-run` covers both policies AND evidence | Pure preview semantics — no side effects on any filesystem | ✓ Good |
| `cpg explain` accepts YAML path or NS/workload | YAML path rejects non-`cpg-` names — guards against explaining non-cpg policies | ✓ Good |
| Session upsert by ID in evidence merge | Required so `finalize()` can rewrite final flow counters | ✓ Good |
