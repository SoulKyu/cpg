  <name>Task 1: Implement Aggregator with temporal flush and LostEvents monitor</name>
  <files>pkg/hubble/aggregator.go, pkg/hubble/aggregator_test.go</files>
  <behavior>
    - TestAggregator_FlushOnTicker: flows accumulate in buckets, ticker triggers flush producing PolicyEvents via BuildPolicy
    - TestAggregator_KeyFromFlow_Ingress: INGRESS flow uses destination endpoint for AggKey
    - TestAggregator_KeyFromFlow_Egress: EGRESS flow uses source endpoint for AggKey
    - TestAggregator_FlushOnChannelClose: when input channel closes, remaining flows are flushed
    - TestAggregator_FlushOnContextCancel: context cancellation triggers final flush before returning
    - TestAggregator_SkipEmptyNamespace: flows with empty namespace on target endpoint are skipped (logged at debug)
    - TestMonitorLostEvents_AggregatesWarnings: lost events are accumulated and logged every 30s, not per-event
    - TestMonitorLostEvents_FinalSummary: on context cancel, logs total lost events if > 0
  </behavior>
  <action>
    Create `pkg/hubble/aggregator.go`:

    ```go
    type AggKey struct {
        Namespace string
        Workload  string
    }

    type Aggregator struct {
        interval time.Duration
        logger   *zap.Logger
    }

    func NewAggregator(interval time.Duration, logger *zap.Logger) *Aggregator
    func (a *Aggregator) Run(ctx context.Context, in <-chan *flowpb.Flow, out chan<- policy.PolicyEvent) error
    ```

    Implementation details:
    - `Run` uses select loop: receives flows, accumulates in `map[AggKey][]*flowpb.Flow` buckets
    - On ticker: call `flush()` which iterates buckets, calls `policy.BuildPolicy()` for each, sends PolicyEvent to out channel
    - On `<-in` closed (ok=false): flush remaining, return nil
    - On `ctx.Done()`: flush remaining, return nil
    - `keyFromFlow`: INGRESS -> destination endpoint, EGRESS -> source endpoint
    - Skip flows where target endpoint has empty Namespace (log debug, avoid empty dir names)
    - `flush` clears buckets after sending (delete loop)
    - `defer close(out)` at start of Run

    Also create `monitorLostEvents` function in same file:
    ```go
    func monitorLostEvents(ctx context.Context, ch <-chan *flowpb.LostEvent, logger *zap.Logger) error
    ```
    - Accumulates `periodLost` and `totalLost` (uint64)
    - 30s ticker: if periodLost > 0, log warn with period and total counts, reset periodLost
    - On ctx.Done(): if totalLost > 0, log final warn
    - On channel close: return nil

    Use `policy.BuildPolicy` from pkg/policy (already tested in Phase 1). Use `labels.WorkloadName` for workload derivation from flow endpoint labels.

    For tests, use the test helpers from `pkg/policy/testdata/` (IngressTCPFlow, EgressUDPFlow) to build realistic flows. Use short ticker intervals (10ms) for fast tests.
  </action>
  <verify>
    <automated>go test ./pkg/hubble/ -run "TestAggregator|TestMonitorLostEvents" -count=1 -v -race</automated>
  </verify>
  <done>Aggregator accumulates flows by (namespace, workload), flushes on ticker/close/cancel, skips empty namespaces. LostEvents monitor aggregates warnings every 30s. All tests pass with race detector.</done>
</task>

    Create `pkg/hubble/pipeline.go`:

    ```go
    type PipelineConfig struct {
        Server        string
        TLSEnabled    bool
        Timeout       time.Duration
        Namespaces    []string
        AllNamespaces bool
        OutputDir     string
        FlushInterval time.Duration
        Logger        *zap.Logger
    }

    type SessionStats struct {
        StartTime       time.Time
        FlowsSeen       uint64
        PoliciesWritten uint64
        LostEvents      uint64
        OutputDir       string
    }

    func RunPipeline(ctx context.Context, cfg PipelineConfig) error
    ```

    Implementation:
    - `RunPipeline` sets up signal.NotifyContext (SIGINT, SIGTERM)
    - Creates Client, Aggregator, output.Writer
    - Calls `client.StreamDroppedFlows(ctx, cfg.Namespaces, cfg.AllNamespaces)`
    - Creates `policies` channel (buffered 64)
    - Uses `errgroup.WithContext` with 3 goroutines:
      1. `agg.Run(ctx, flows, policies)` -- aggregation
      2. `writePolicies(ctx, policies, writer, stats, logger)` -- write loop
      3. `monitorLostEvents(ctx, lostEvents, stats, logger)` -- lost events
    - After `g.Wait()`, logs SessionStats
    - `writePolicies` helper: range over policies channel, call `writer.Write()`, increment stats

    For tests, define an interface for the streaming source so pipeline can be tested with a mock:
    ```go

        StreamDroppedFlows(ctx context.Context, namespaces []string, allNS bool) (<-chan *flowpb.Flow, <-chan *flowpb.LostEvent, error)
    }
    ```
    Make `RunPipeline` accept `FlowSource` internally or provide a `RunPipelineWithSource` test helper. Alternatively, create a helper `runPipelineInternal` that accepts channels directly.
