    - TestBuildFilters_SingleNamespace: when namespaces=["production"], returns two FlowFilter entries (OR-ed): one with SourcePod=["production/"] and one with DestinationPod=["production/"], both with Verdict DROPPED
    - TestBuildFilters_MultipleNamespaces: when namespaces=["prod","staging"], both filters have both prefixes ["prod/","staging/"]
    - TestBuildFilters_EmptyNamespaces: when namespaces=nil and allNS=false, behaves like allNS (single filter, verdict only)
  </behavior>
  <action>
    Create `pkg/hubble/client.go` with package declaration, imports, and `buildFilters(namespaces []string, allNS bool) []*flowpb.FlowFilter` function.

    Key implementation details from research:
    - FlowFilter fields within a single filter are AND-ed; multiple whitelist filters are OR-ed
    - Namespace prefix format: `ns + "/"` (trailing slash required to avoid "prod" matching "production")
    - Two separate FlowFilter entries needed: one for SourcePod, one for DestinationPod
    - Both include `Verdict: []flowpb.Verdict{flowpb.Verdict_DROPPED}`

    Also stub the `Client` struct and `NewClient` constructor (no implementation yet, just the type signature) so the file compiles. RED tests first, then GREEN implementation.

    Before building, promote gRPC to direct dependency:
    ```bash
    go get google.golang.org/grpc
    go get golang.org/x/sync
    ```
  </action>

  </verify>
  <done>All 4 buildFilters tests pass. FlowFilter construction handles all-namespaces, single namespace, multiple namespaces, and empty namespace cases correctly.</done>

    Implement the full `Client` struct in `pkg/hubble/client.go`:

    ```go
    type Client struct {

    }
    ```

    `NewClient(server string, tlsEnabled bool, timeout time.Duration, logger *zap.Logger) *Client`

    `StreamDroppedFlows(ctx context.Context, namespaces []string, allNS bool) (<-chan *flowpb.Flow, <-chan *flowpb.LostEvent, error)`:
    - Creates gRPC connection with `grpc.NewClient()` + `insecure.NewCredentials()` (or TLS if enabled using `credentials.NewTLS(&tls.Config{})`)
    - Creates `observer.NewObserverClient(conn)`
    - Calls `client.GetFlows(ctx, req)` with `Follow: true` and filters from `buildFilters`
    - Spawns goroutine that reads from stream, dispatches to `flows` or `lostEvents` channels
    - Goroutine closes both channels and connection on stream end or context cancel
    - Use buffered channels: flows=256, lostEvents=16

    For testing, use an interface-based mock approach:
    - Define a `flowStream` interface matching `observer.Observer_GetFlowsClient` (just `Recv()` and `Context()`)
    - Create a `mockStream` that returns pre-configured responses
    - Test the internal streaming loop logic via a helper that accepts the interface

    Important: Use `select` with `ctx.Done()` on channel sends to prevent goroutine leaks (anti-pattern from research).
