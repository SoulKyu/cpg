## HTTP L7 Generation

- [x] **HTTP-01**: When `--l7` is set and `Flow.L7.Http` records are present for a (src, dst, port) tuple, cpg generates a `toPorts.rules.http` block alongside the L4 port rule. — Phase 8
- [x] **HTTP-02**: HTTP method is normalized to uppercase before emission. — Phase 8
- [x] **HTTP-03**: HTTP path is emitted as a Cilium-compatible RE2 regex via `regexp.QuoteMeta` + `^…$` anchoring. — Phase 8
- [x] **HTTP-04**: Multiple distinct (method, path) observations for the same (src, dst, port) tuple merge into a single `toPorts.rules.http` list. — Phase 8
- [x] **HTTP-05**: HTTP `Headers`, `Host`, and `HostExact` rules are NOT generated (anti-feature, secret-leak risk). — Phase 8

## DNS L7 Generation

- [x] **DNS-01**: When `--l7` is set and `Flow.L7.Dns` records are present for an egress dropped flow, cpg generates an egress rule with `toFQDNs.matchName` (literal hostname) paired with `toPorts.rules.dns.matchName`. — Phase 9
- [x] **DNS-02**: For every CNP that contains `toFQDNs`, cpg automatically emits a companion egress rule allowing UDP+TCP/53 to kube-dns (selector hardcoded for v1.2). — Phase 9
- [x] **DNS-03**: DNS `matchPattern` (glob) is NOT auto-generated — only `matchName` (literal). — Phase 9
- [x] **DNS-04**: When `Flow.L7.Dns` is absent, cpg falls back to the existing v1.1 CIDR-based egress rule, byte-identical to v1.1. — Phase 9
