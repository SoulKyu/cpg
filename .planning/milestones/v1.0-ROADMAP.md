# Milestone v1.0 — MVP (Core Policy Generator)

**Status:** ✅ Shipped
**Date:** 2026-03-08
**Phases:** 1 → 3
**Plans:** 7 total (3 + 2 + 2)

## Delivered

A Go CLI (`cpg generate`) that connects to Hubble Relay via gRPC, observes dropped flows in real-time, and produces ready-to-apply CiliumNetworkPolicy YAML — with smart label selection, CIDR rules for external traffic, file + cluster deduplication, and auto port-forward for zero-friction local use.

## Key Accomplishments

1. **Correct ingress/egress CNP generation** from dropped flows with exact port + protocol and smart `app.kubernetes.io/*` label selectors (Phase 1).
2. **Live Hubble streaming pipeline** (gRPC → FlowFilter → aggregator → policy writer) with namespace filtering and LostEvents overflow warnings (Phase 2).
3. **CIDR-based rules for world identity** (external traffic) replacing raw endpoint selectors (Phase 3).
4. **Deduplication at three layers** — file-on-disk, cross-flush in-session, and live cluster via client-go (Phase 3).
5. **Auto port-forward to hubble-relay** via SPDY for zero-config local use (Phase 3).
6. **Structured logging** with zap, full session stats, graceful shutdown via errgroup (Phases 2/3).

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| gRPC-only initially | Simplified v1 architecture, native typed flow data | ✓ Good — offline replay added in v1.1 |
| Smart label selection (app.kubernetes.io/*, workload) | Generated policies remain valid across rescheduling | ✓ Good |
| One YAML file per policy | git-diff friendly, selective kubectl apply | ✓ Good |
| Exact ports over named ports | No ambiguity, matches flow data directly | ✓ Good |
| Three-layer dedup (file + cross-flush + cluster) | Prevents duplicates in all generation scenarios | ✓ Good |
| CIDR rules for `world` identity | Endpoint selectors don't apply to external traffic | ✓ Good |
| Auto port-forward via SPDY | UX parity with `hubble` CLI, zero manual setup | ✓ Good |
