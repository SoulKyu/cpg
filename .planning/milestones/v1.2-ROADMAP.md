# Milestone v1.2 — L7 Policies (HTTP + DNS)

**Status:** ✅ Shipped
**Date:** 2026-04-25
**Phases:** 7 → 9
**Plans:** 12 total (4 + 4 + 4)
**Tests:** 319 across 9 packages (up from 180 at end of v1.1)

## Delivered

cpg now generates L7 CiliumNetworkPolicies (HTTP method+path, DNS FQDN matchName) from observed Hubble L7 flows when the user passes `--l7`. The two-step workflow (deploy L4 → enable L7 visibility → re-run with `--l7`) is documented and the CLI guides users with pre-flight cluster checks, single-warning empty-records detection, and a starter visibility CNP. `cpg explain` surfaces L7 attribution per rule with three new exact-match filters. Default behavior without `--l7` is byte-identical to v1.1.

## Key Accomplishments

1. **L7 infrastructure prep (Phase 7)** — fixed a latent `mergePortRules` Rules-field-drop bug (silent L7 data loss waiting to happen), bumped evidence schema v1 → v2 with `L7Ref` field, added pre-flight K8s checks (`cilium-config.enable-l7-proxy=true` + `cilium-envoy` DaemonSet) with warn-and-proceed semantics, plumbed `--l7` and `--no-l7-preflight` flags. Zero user-visible behavior change at end of phase.
2. **HTTP L7 generation (Phase 8)** — `Flow.L7.Http` records produce `toPorts.rules.http` blocks with method (uppercase normalized) + path (`regexp.QuoteMeta` + `^…$` anchored). Multi-(method, path) merging into one PortRule. `headerMatches`/`host`/`hostExact` NEVER emitted (anti-feature). VIS-01 single-warning when `--l7` set but zero L7 records arrive.
3. **DNS L7 generation (Phase 9)** — `Flow.L7.Dns` queries produce `toFQDNs.matchName` (literal, trailing-dot stripped) paired with `toPorts.rules.dns.matchName`. Mandatory companion DNS-53 rule auto-injected for every CNP containing `toFQDNs` (idempotent post-process). No `matchPattern` glob in v1.2.
4. **`cpg explain` L7 surface (Phase 9)** — three new exact-match filters: `--http-method`, `--http-path`, `--dns-pattern`. AND-combined. Renderer extension across text/JSON/YAML formats with `L7: HTTP <M> <P>` or `L7: DNS <name>` lines.
5. **README two-step workflow + starter L7-visibility CNP (Phase 9)** — full `#l7-prerequisites` section documenting the deploy-L4 → enable-visibility → re-run-with-`--l7` flow, three ways to enable visibility (proxy-visibility annotation, bootstrap CNP, cluster-wide enable-l7-proxy), copy-pasteable starter CNP.
6. **Evidence schema v2 with no back-compat** — clean cut, reader rejects `schema_version != 2` with the literal string `$XDG_CACHE_HOME/cpg/evidence/` in the error (incident-response grep-bait). v1.1 shipped 2026-04-24, no production caches.
7. **Byte-stability invariant** — `--l7=false` against any v1.1 fixture produces byte-identical YAML to v1.1 baseline. Tested end-to-end across all three phases.

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| `--l7` opt-in (default OFF) | Preserves v1.1 behavior for users who didn't ask for L7; avoids silently activating L7 when records appear | ✓ Good |
| No regex/glob inference (paths, FQDNs) | Auto-collapse hallucinates allow-lists broader than observed; honest verbose output beats clever broad output | ✓ Good — defer to v1.3 |
| HTTP path as `regexp.QuoteMeta` + `^…$` | Cilium HTTP path is RE2 regex; failure to escape allows broader match than observed; failure to anchor matches substrings | ✓ Good |
| HTTP `headerMatches`/`host`/`hostExact` NEVER emitted | Risk of leaking `Authorization` / `Cookie` / session tokens into committed YAML | ✓ Good — anti-feature documented in README |
| Mandatory kube-dns companion rule for every `toFQDNs` | ToFQDNs requires DNS resolution to learn IPs; without companion, policy applies but DNS denied → silent total breakage | ✓ Good — invariant test enforced |
| Evidence schema v2 with no back-compat | v1.1 shipped 2026-04-24, no production caches; clean cut keeps reader simple | ✓ Good — wipe instruction in error message |
| Pre-flight checks warn-and-proceed (no abort) | Operators with reduced K8s permissions (CI service accounts) must not be locked out | ✓ Good |
| `--no-l7-preflight` flag | CI restricted-RBAC, kubeconfig without kube-system access, air-gapped use | ✓ Good |
| AggKey does NOT extend with L7 | L7 is a property of port-rule inside bucket, not of bucket; extending AggKey would shatter buckets | ✓ Good — pipeline structure unchanged |
| Companion rule selector hardcoded `k8s-app=kube-dns` | Auto-detection across CNI distributions adds complexity without v1.2 value | — Deferred to v1.3 (DNS-FUT-02) |
| DROPPED-only verdict filter (kept) | REDIRECTED means Cilium PROXIED; new rules from already-policied traffic would be wrong | ✓ Good — REFUSED gap deferred to v1.3 (L7-FUT-01) |
| Single VIS-01 warning per pipeline run | Multiple workloads observed → one warning, not N | ✓ Good — sync.Once / Finalize-once pattern |

## Technical Debt / Follow-ups

- kube-dns selector autodetection across CNI distributions (EKS / GKE / AKS / vanilla) — deferred to v1.3 (DNS-FUT-02). Hardcoded `k8s-app=kube-dns` with YAML comment is acceptable for v1.2 scope.
- `--include-l7-forwarded` for DNS REFUSED denials surfaced as `Verdict_FORWARDED` — deferred to v1.3 (L7-FUT-01). Live-cluster validation needed to size the actual gap.
- Path / FQDN auto-collapse heuristics — deferred to v1.3 (HTTP-FUT-01, DNS-FUT-01) pending user feedback on rule explosion.
- ToFQDNs from IP→name correlation when DNS records are missed but resolved IP is present — deferred to v1.3 (DNS-FUT-03).
- `--min-flows-per-l7-rule N` low-confidence gate — deferred to v1.3 (L7-FUT-02).
